IIS 6.0 PUT上传漏洞+iis解析漏洞getshell

渗透测试
  • IIS

IIS 6.0 PUT上传漏洞+iis解析漏洞getshell。环境为本地靶场环境。

IIS 6.0 PUT上传漏洞。产生原因是IIS Server在Web服务扩展中开启了WebDAV。并且管理员在配置权限时，没有遵循最小原则。给IIS配置了可以写入的权限，包括网站根目录。

  • 靶场环境

    • window2003+iis6.0+asp环境

  • 攻击机环境

    • window xp 工具 iisputscaner + iiswrite

  • 复现步骤

用iisputscaner查看一下目标网的IP，PUT显示为YES及代表目网站开启了WebDAV。

然后运行桂林老兵的iiswrite工具，选中OPTTONS,然后提交数据包。有如下返回内容代表存在put上传漏洞

选到PUT选项卡，请求文件名为你上传到服务器上的文件名（只能是TXT文件）把小马也改成TXT文件，然后提交数据包上传。
返回201代表上传成功

但是这个TXT文件是无法被iis解析的，所以要利用到的是copy，主要目的是为了将txt的文件修改为asp的，从而可以将文件变成可执行的脚本文件。

然后发现直接改成asp文件，提交数据不成功，一直没有回显，想到IIS还有一个解析漏洞。于是把文件名改成 shell.asp;.jpg

可以看到返回值201，上传成功了

再次访问 http://192.168.1.102/shell.asp;.jpg

可以看到文件上传成功了
尝试用菜刀连接
